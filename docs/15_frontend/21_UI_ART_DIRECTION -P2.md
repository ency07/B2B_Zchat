# 19. PALETA CROMÁTICA OFICIAL

La paleta del sistema debe transmitir ingeniería, precisión y estabilidad.

Nunca debe llamar más la atención que la información.

Los colores existen para comunicar estados, no para decorar.

---

## Fondo Principal

Color extremadamente oscuro.

Debe recordar un panel industrial moderno.

Nunca completamente negro (#000000).

Siempre utilizar un gris muy profundo.

Ejemplo conceptual:

* Graphite
* Gunmetal
* Charcoal

---

## Fondo Secundario

Ligeramente más claro que el fondo principal.

Debe permitir diferenciar zonas de trabajo sin necesidad de bordes pesados.

---

## Paneles

Los paneles nunca deben parecer tarjetas flotantes.

Deben sentirse como superficies de trabajo.

Características:

* Elevación mínima.
* Bordes discretos.
* Contraste bajo.
* Mucho espacio interno.

---

## Color Primario

Azul industrial.

Debe utilizarse únicamente para:

* acciones principales
* enlaces importantes
* elementos activos
* navegación
* indicadores positivos

Nunca utilizarlo para decorar.

---

## Color Secundario

Grises técnicos.

Son los colores predominantes.

El azul debe ocupar menos del 10% de la interfaz.

---

## Colores Semánticos

Verde

Solo:

confirmaciones

estados correctos

procesos finalizados

---

Amarillo

Solo:

advertencias

procesos pendientes

alertas

---

Rojo

Solo:

errores

riesgos

alarmas

eliminaciones

---

Morado

Reservado para:

IA

automatización

predicciones

machine learning

---

Turquesa

Reservado para:

ingeniería

CFD

simulación

análisis técnicos

---

Nunca crear colores nuevos para cada módulo.

Toda la plataforma debe compartir exactamente el mismo lenguaje cromático.

---

# 20. CONTRASTE

El contraste debe ser alto para texto.

Pero moderado para componentes.

Nunca utilizar:

blanco puro sobre negro puro.

Debe existir suavidad visual.

---

# 21. BORDES

Los bordes deben ser extremadamente discretos.

No deben competir con el contenido.

El usuario casi no debe percibirlos.

Los bordes sirven únicamente para separar información.

Nunca para decorar.

---

# 22. ESQUINAS

Todo el sistema debe compartir exactamente el mismo radio.

No mezclar:

4px

6px

8px

12px

16px

Elegir una escala consistente.

Nunca utilizar esquinas completamente redondas tipo aplicación móvil.

Debe sentirse empresarial.

---

# 23. SOMBRAS

Las sombras deben ser casi imperceptibles.

Prohibido:

Glow.

Neón.

Sombras enormes.

Blur exagerado.

Sombras difusas.

El usuario nunca debe pensar:

"Qué bonita esa sombra."

Debe ni siquiera notarla.

---

# 24. PROFUNDIDAD

La profundidad debe lograrse mediante:

espaciado

alineación

contraste

jerarquía

No mediante efectos.

---

# 25. TIPOGRAFÍA

La tipografía es uno de los elementos más importantes del sistema.

Debe transmitir ingeniería.

No moda.

No creatividad.

---

## Fuentes permitidas

Inter

Outfit

IBM Plex Sans

Geist

Nunca mezclar más de dos familias tipográficas.

---

## Pesos

Utilizar únicamente los necesarios.

Regular

Medium

Semibold

Bold

Evitar Black excepto para Hero.

---

## Alineación

Todo el contenido empresarial debe alinearse a la izquierda.

Nunca centrar grandes bloques de texto.

Solo:

Hero principal

Pantallas vacías

Confirmaciones

Landing

---

# 26. ICONOGRAFÍA

Únicamente se permite:

Lucide Icons.

Todos los iconos deben compartir:

mismo grosor

mismo tamaño

misma alineación

mismo estilo

Nunca mezclar librerías.

Nunca utilizar emojis.

Nunca utilizar iconos rellenos.

Nunca utilizar ilustraciones infantiles.

---

# 27. ILUSTRACIONES

Las ilustraciones deben ser mínimas.

Preferiblemente:

diagramas

planos

líneas técnicas

wireframes

esquemas

Nunca personajes.

Nunca dibujos caricaturescos.

Nunca mascotitas.

---

# 28. FOTOGRAFÍA

Las fotografías deben mostrar:

plantas industriales

turbinas

ductos

ventiladores

soldadura

ingenieros trabajando

CNC

corte láser

fabricación

acero

metal

estructuras industriales

instrumentación

Nunca utilizar:

personas sonriendo frente a un portátil

reuniones corporativas

cafeterías

oficinas modernas

call centers

equipos celebrando

---

# 29. RENDERS

Los renders deben parecer:

Autodesk

SolidWorks

Fusion 360

Inventor

No arte conceptual.

No ilustraciones.

No renders exagerados.

---

# 30. FONDOS

El fondo jamás debe competir con el contenido.

Nunca utilizar:

texturas

patrones

ruido

formas abstractas gigantes

ondas

gradientes llamativos

Si existe un fondo decorativo debe ser casi invisible.

---

# 31. ANIMACIONES

Toda animación debe tener propósito.

Jamás decorar.

Las animaciones existen para:

explicar

guiar

confirmar

orientar

Nunca para impresionar.

---

## Duración

150 ms

200 ms

250 ms

Máximo 300 ms.

Nunca rebotes.

Nunca elasticidad.

Nunca efectos teatrales.

---

# 32. MICROINTERACCIONES

Permitidas:

Hover

Focus

Pressed

Loading

Skeleton

Fade

Slide corto

Cambio de estado

No más.

---

# 33. TRANSICIONES

Las transiciones deben sentirse naturales.

No deben ser percibidas como una animación.

Deben sentirse como continuidad del trabajo.

---

# 34. LOADING

Nunca utilizar spinners gigantes.

Preferir:

Skeleton

Placeholder

Carga progresiva

Streaming

Server Components

La interfaz nunca debe quedarse vacía esperando.

---

# 35. VACÍOS

Los estados vacíos deben enseñar.

Nunca simplemente decir:

"No hay datos."

Deben indicar:

qué ocurrió

cómo solucionarlo

qué hacer después

---
