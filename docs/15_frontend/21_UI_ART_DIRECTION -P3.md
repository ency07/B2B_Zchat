# 36. DIRECCIÓN ARTÍSTICA DE LOS DASHBOARDS

Los dashboards nunca serán una colección de tarjetas.

Un dashboard debe contar una historia.

Debe responder preguntas.

Debe permitir tomar decisiones.

Cada widget debe existir porque ayuda a decidir algo.

Si un widget no ayuda a tomar decisiones, debe eliminarse.

---

# 37. UN DASHBOARD NO ES UN CATÁLOGO DE KPIs

Queda prohibido construir pantallas como:

□ Ventas

□ Clientes

□ Productos

□ Proyectos

□ Usuarios

□ Facturas

sin ninguna relación entre sí.

Eso no es inteligencia.

Eso es decoración.

Los KPIs deben tener contexto.

Ejemplo:

Ventas del mes

↓

Comparación con el mes anterior

↓

Variación porcentual

↓

Motivo del cambio

↓

Clientes responsables

↓

Acción recomendada

El dashboard debe explicar.

No solamente mostrar.

---

# 38. CADA PANTALLA DEBE TENER UN OBJETIVO

Antes de diseñar una página el sistema debe responder:

¿Qué decisión permitirá tomar esta pantalla?

Si no existe respuesta:

la pantalla está mal diseñada.

# 40. LOS KPIs

Los KPIs nunca son decoración.

Cada KPI debe responder:

¿Qué mide?

¿Respecto a qué?

¿Cómo va?

¿Qué significa?

¿Qué acción genera?

Si no responde esas preguntas:

el KPI sobra.

---

# 41. LOS GRÁFICOS

Un gráfico nunca se coloca porque "queda bonito".

Debe existir únicamente cuando la información sea más fácil de entender visualmente.

Si una tabla comunica mejor...

usar tabla.

---
