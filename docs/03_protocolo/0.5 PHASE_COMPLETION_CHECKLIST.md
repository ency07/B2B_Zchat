# ERP B2B Premium - Phase Completion Checklist

This checklist must be reviewed and marked as completed at the end of each development phase to verify technical and quality standards.

## Quality Gates

### 1. Database & Schema Gate
- [ ] DDL migration file created in `supabase/migrations/` with correct sequence number.
- [ ] Foreign keys, references, and `CHECK` constraints defined.
- [ ] No syntax errors in SQL triggers or functions.
- [ ] Performance indexes created on all filterable columns.

### 2. Security Gate (RLS & Isolation)
- [ ] RLS enabled on all newly created tables.
- [ ] Multi-tenant isolation policy (`tenant_id = user_tenant`) applied.
- [ ] No cross-tenant visual leaks or operations permitted.

### 3. Traceability Gate
- [ ] Incremental sequence prefix assigned to tables via `tenant_sequences` trigger.
- [ ] Audit logs trigger (`process_audit_log`) active on all tables.
- [ ] Traceability fields trigger (`handle_approval_traceability`) active on all tables.
- [ ] Physical deletes blocked, soft delete fields implemented.
- [ ] Cancellation checks (minimum 10 characters for reasons) verified.

### 4. Code & Validation Gate
- [ ] TypeScript test script created in `scripts/`.
- [ ] Command added to `package.json`.
- [ ] Tests run and completed successfully without compile or runtime errors.

### 5. Documentation Gate
- [ ] `docs/00_GOVERNANCE/PROJECT_MEMORY.md` updated with the decisions and schema changes of the phase.
- [ ] `docs/MASTER_INDEX.md` updated with new files.
- [ ] Implementation report created in the workspace root.
- [ ] Walkthrough and task logs updated in the artifacts directory.
