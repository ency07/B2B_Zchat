# ERP B2B Premium - AI Execution Constitution

This constitution governs the behavior, design patterns, and execution principles of the AI coding assistant (Antigravity) throughout the lifecycle of this project.

## Core Principles

### I. Multi-Tenant Isolation (NON-NEGOTIABLE)
*   **Aisolation:** Every table (with no exceptions except global platform configuration tables like `tenants`) must have a `tenant_id` column.
*   **Row-Level Security (RLS):** RLS must be enabled on every table, with policies restricting read and write access to the user's active tenant.
*   **No Cross-Tenant Queries:** All transactions, functions, and triggers must operate within the boundary of a single tenant.

### II. Spec-Driven & No Invention (NON-NEGOTIABLE)
*   **No Invention Policy:** The AI is strictly prohibited from inventing fields, tables, states, rules, relationships, or permissions.
*   **Verification Waterfall:** Before asking any question, the AI must check `0.2 DECISIONES GLOBALES CONGELADAS.md`, `PROJECT_MEMORY.md`, and the dictionary of data. If the answer exists, it must be applied automatically without asking.
*   **Discovery Protocol:** Gaps must be documented in a validation table, and only new, module-specific rules may be presented as legitimate questions.

### III. Physical Delete Prohibition
*   **Soft Delete Only:** Every operational table must include soft delete columns: `deleted_at` (timestamp), `deleted_by` (uuid), and `delete_reason` (text).
*   **Trigger Enforcement:** A database trigger must block all physical `DELETE` statements on operational tables, raising a SQL exception.

### IV. Full Audit & Traceability
*   **Audit Logging:** Every insert, update, or soft delete must be registered automatically in the global `audit_log` table via a database trigger.
*   **Traceability Fields:** Every record must track creation and update metadata (`created_by`, `created_at`, `updated_by`, `updated_at`).
*   **Justified Cancellations:** Changing status to `'ANULADA'` or `'CANCELADA'` must require a `cancel_reason` of at least 10 characters, validated by trigger.
