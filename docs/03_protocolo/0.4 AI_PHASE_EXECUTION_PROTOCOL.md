# ERP B2B Premium - AI Phase Execution Protocol

This document defines the mandatory step-by-step workflow that the AI must follow when executing any development phase.

## Phase Execution Lifecycle

### Step 1: Research & Validation
1.  Read the functional requirements of the target phase in `docs/05_modelo_funcional/` and `docs/08_datos/`.
2.  Perform the **Validation Table** audit against:
    *   `0.2 DECISIONES GLOBALES CONGELADAS.md`
    *   `PROJECT_MEMORY.md`
    *   `0.3 AI_EXECUTION_CONSTITUTION.md`
3.  Any global rule or convention found must be classified as **APLICAR** (apply automatically). Only unresolved module-specific details are classified as **PREGUNTAR**.

### Step 2: Implementation Plan
1.  Create or update the implementation plan (`implementation_plan.md`) in the artifacts directory.
2.  Ensure it includes:
    *   **Sección A:** Decisions inherited and applied automatically.
    *   **Sección B:** Proposed DDL schemas and database triggers.
    *   **Sección C:** Legitimate open questions (if any).
    *   **Sección D:** Verification plan.
3.  Stop and obtain explicit user approval before writing any code or migrations.

### Step 3: DDL Migration Code
1.  Create the SQL migration file in `supabase/migrations/` using the sequential naming convention (e.g., `202606170000XX_phase_name.sql`).
2.  Write clean SQL constraints, foreign keys, and indexes.
3.  Implement standard triggers:
    *   Autogenerate correlatives (`tenant_sequences`).
    *   Enforce audit logs (`process_audit_log`).
    *   Enforce traceability (`handle_approval_traceability`).
    *   Block physical deletes (`block_physical_delete`).
    *   Enforce cancellation checks.

### Step 4: Verification Testing
1.  Write a TypeScript validation script under `scripts/test-phase_name.ts`.
2.  Run the tests locally and ensure it validates schemas, triggers, RLS, and security constraints.
3.  Fix all errors, lints, or environment issues before proceeding.

### Step 5: Closure & Logging
1.  Create `FASE_XX_IMPLEMENTATION_REPORT.md` in the workspace root.
2.  Update `walkthrough.md` in the artifacts folder.
3.  Mark tasks as completed in `task.md`.
