# ROADMAP DE CONFIGURACIÓN GLOBAL (FASES 31–34)

Este documento detalla el plan incremental para el Centro de Configuración Global del ERP B2B Premium, diseñado para evitar valores fijos (hardcoded) y posibilitar la personalización por empresa sin alteración de código.

---

## FASE 31: Centro de Configuración Empresarial (Settings)
**Objetivo:** Crear el repositorio central de datos corporativos de cada inquilino.
*   **Empresa:** Información legal (Nombre comercial, Razón Social, NIT, Régimen tributario, Dirección, Datos de contacto).
*   **Localización:** Zona horaria, Horario laboral, Idioma, Moneda base, Separador decimal, Formato de fecha y hora.
*   **Identidad Base:** URLs de logotipos (Claro, Oscuro, PDF, Login), Favicon, Tipografías, Border Radius, Sombras.
*   **Formatos de Documentos:** Encabezados, Pies de página, Ubicación de firmas, Sellos, Márgenes, Generador de código QR para PDFs.
*   **Configuración del ERP:** Nombre visible del sistema, Versión expuesta, Enlaces de soporte, Manual de usuario, Enlaces legales (Privacidad, Términos, Cookies).

---

## FASE 32: White Label (Branding Dinámico)
**Objetivo:** Permitir la personalización visual extrema del ERP por inquilino.
*   **Interfaz Dinámica:** Aplicación dinámica de colores primario/secundario/estado, fuentes, sombras y transiciones en la barra lateral, menús y widgets según la configuración cargada del tenant.
*   **Personalización de Pantallas:** Pantalla de Login, Carga, páginas de error (404/500) personalizadas.
*   **Aplicación SaaS Extrema:** Posibilitar que la Empresa A (ej: Siemens ERP) y la Empresa B (ej: ABB Service Cloud) vean interfaces visualmente distintas, incluyendo correos y PDFs con branding exclusivo, sin necesidad de redistribuir código.

---

## FASE 33: Integraciones y Canales
**Objetivo:** Mover todas las variables operativas de entorno (.env) a base de datos de manera segura.
*   **Gateways de Correo:** Configuración dinámica de proveedores de email (SMTP, Resend, Sendgrid, SES, Mailgun).
*   **Mensajería y Chat:** Configuración por tenant de WhatsApp (Twilio, Meta, 360dialog), Telegram (Bot Token, Chat IDs, Grupos) y SMS (Twilio, Vonage).
*   **Servicios Externos:** Credenciales de mapas (Google Maps, Mapbox), Calendarios (Google, Microsoft) y APIs de Inteligencia Artificial (OpenAI, Anthropic, Gemini, DeepSeek).
*   **Pasarelas de Pago:** Stripe, MercadoPago, PayPal, Wompi.
*   **Almacenamiento (Storage):** Supabase Storage, S3, Azure Blob, Backblaze, Cloudflare R2.
*   **Webhooks:** Gestión de webhooks entrantes y salientes configurables.
*   **Enrutamiento Telefónico:** Centralización de números de contacto específicos (Ventas, Soporte, Emergencias, Facturación) e inmutabilidad de variables telefónicas (ningún número debe estar fijo en código).

---

## FASE 34: Administración Avanzada
**Objetivo:** Otorgar control absoluto al administrador del tenant sin tocar base de datos física.
*   **Perfiles de Usuario:** Avatar, idioma de preferencia, tema preferido, firma escaneada.
*   **Mapeo de Menús y Módulos:** Habilitar/deshabilitar secciones del ERP y renombrar módulos en la UI (ej: cambiar "Trabajos" a "Órdenes de Servicio" o "Clientes" a "Cuentas").
*   **Campos Personalizados (Custom Fields):** Permitir añadir campos dinámicos (Texto, Número, Fecha, Lista de opciones, Archivo, Boolean, JSON) a cualquier tabla principal (clients, jobs, requirements, etc.) de forma dinámica.
*   **Motor de Automatización:** Definición visual de reglas "Si X Evento ocurre, despachar acción Y" (ej: Si Factura Vence $\rightarrow$ Enviar Telegram a Gerente $\rightarrow$ Crear Tarea de Cobranza $\rightarrow$ Enviar WhatsApp a Cliente).
