# CAPÍTULO 10

# ARQUITECTURA TÉCNICA

## OBJETIVO

Definir cómo debe construirse técnicamente el sistema.

No define funcionalidades.

Define estructura, capas, dependencias y restricciones.

---

# PRINCIPIO 1

Arquitectura modular.

Prohibido sistema monolítico caótico.

Cada módulo debe ser independiente.

---

# CAPAS

## PRESENTACIÓN

Frontend.

Responsable de:

* Visualización
* Formularios
* Navegación
* Dashboards

No contiene reglas de negocio.

---

## API

Backend.

Responsable de:

* Exponer servicios
* Validaciones
* Autorización
* Orquestación

---

## DOMINIO

Responsable de:

* Reglas negocio
* Casos de uso
* Estados
* Transiciones
* Aprobaciones

Toda lógica crítica vive aquí.

---

## PERSISTENCIA

Responsable de:

* Base de datos
* Repositorios
* Consultas

No contiene reglas de negocio.

---

# REGLA FUNDAMENTAL

```text
UI
 ↓
API
 ↓
Dominio
 ↓
Repositorio
 ↓
Base Datos
```

Nunca:

```text
UI → Base Datos

UI → SQL

Frontend → Supabase directo
```

para operaciones de negocio.

# MULTITENANCY

Todo registro debe tener:

```text
tenant_id
```

Obligatorio.

---

## Restricción

Toda consulta debe filtrar:

```sql
WHERE tenant_id = ?
```

Sin excepción.

---

# BASE DE DATOS

## Convenciones

Tablas:

```text
snake_case
plural
```

Ejemplos:

```text
clients

requirements

quotes

jobs
```

---

## Campos

```text
id uuid

tenant_id uuid

created_at

created_by

updated_at

updated_by
```

---

## Soft Delete

Todas las tablas operativas:

```text
deleted_at

deleted_by
```

---

## Restricción

Prohibido DELETE físico.

---

# EVENTOS

Toda operación crítica genera evento.

Ejemplos:

```text
ClientCreated

QuoteApproved

JobClosed

InvoicePaid
```

---

# AUDITORÍA

Toda operación crítica:

```text
CREATE

UPDATE

DELETE_SOFT

STATUS_CHANGE

APPROVE

REJECT
```

debe registrar auditoría.

# DOCUMENTOS

Los archivos NO se almacenan en tablas.

Guardar:

```text
path

url

metadata
```

Base de datos.

Archivo real en Storage.

---

# CONFIGURACIÓN

Prohibido:

```text

Flujos hardcodeados

Catálogos hardcodeados
```

Todo configurable.

---

